# 购物中心收益测算软件

一个面向城市更新和存量商业资产改造的前期收益判断工具。

本项目用于帮助业主、投资人、设计公司或运营方，在早期快速判断一个老购物中心、存量商业资产或城市更新项目是否值得继续推进。

## 当前版本

- 版本：v1.0 原型版
- 日期：2026-05-29
- 形态：本地可运行的静态网页原型
- 仓库：https://github.com/byrandna/codex-test

当前版本已经完成第一版可演示原型，重点验证“输入项目条件 -> 测算收益和成本 -> 输出 ROI、回收期和推进建议”的产品逻辑。

## 这个软件解决什么问题

在存量商业改造早期，很多项目还没有进入正式设计、造价、招商和融资阶段，但决策者已经需要回答：

- 这个项目是否值得继续看；
- 改造后大概能产生多少租金；
- 改造大概要花多少钱；
- 每年可能产生多少 NOI；
- 投资回报率大概是多少；
- 多久能回本；
- 应该推进、谨慎推进，还是暂不建议推进。

本软件的目标是做一个“投前五分钟判断工具”，帮助用户先判断方向，再决定是否进入更专业的设计、造价、运营和融资流程。

## v1.0 已实现功能

### 1. 项目基础输入

用户可以输入或调整项目名称、片区、建筑面积、可出租比例、当前客流、当前出租率、收购成本、改造成本、其他收入和运营成本比例等基础信息。

### 2. 三层测算结构

当前版本把测算分成三个层级：

- 一级：快速判断，用少量核心参数判断项目大方向；
- 二级：AI 收入测算，加入城市、人口、商圈、交通、竞品、业态和品牌因子；
- 三级：方案成本测算，加入建筑面积、公区、停车、轻改、重改、外立面和机电等工程量因素。

这种结构适合从粗判到细化逐步推进，不要求用户一开始就准备完整资料。

### 3. 租金收入预测

当前版本会综合以下因素估算有效租金：

- 城市租金基准；
- 人口流动；
- 客群结构；
- 商圈成熟度；
- 未来片区发展；
- 产业和办公支撑；
- 周边竞品数量；
- 地铁和道路条件；
- 展示面和可见性；
- 主力品牌或锚点；
- 业态组合；
- 当前客流强度；
- 人工租金调整项。

输出包括市场基准租金、有效租金、年租金收入、可出租面积、租金条件评分和租金信心判断。

### 4. 改造成本估算

当前版本可以估算：

- 收购成本；
- 轻改造成本；
- 重改造成本；
- 停车和外立面成本；
- 机电改造成本；
- 其他投入；
- 总投资额。

成本模型用于早期判断，不替代正式造价清单或工程预算。

### 5. 财务指标测算

当前版本可以输出：

- 年租金收入；
- NOI；
- 总投资；
- ROI；
- 投资回收期；
- 项目评级；
- 推进建议。

ROI 和回收期是当前版本最核心的决策指标。

### 6. AI Agent 风格判断

软件会根据测算结果生成一段类似顾问判断的文字报告，包括：

- 是否建议推进；
- 支撑推进的理由；
- 主要风险；
- 收入端判断；
- 成本端判断；
- 财务端判断；
- 下一步建议。

这部分的目标是让软件不仅给数字，还能解释为什么。

### 7. 敏感性分析

当前版本包含基础敏感性分析，用来观察关键假设变化后 ROI 的变化，例如租金、出租率、改造成本、收购成本和运营成本变化。

## 当前版本边界

v1.0 是原型版，不是正式评估系统。当前还没有实现：

- 真实地图、商圈、人口和租金数据库接入；
- AI 自动读取项目资料；
- 真实竞品数据抓取；
- PDF 或正式报告导出；
- 用户账号和项目保存；
- 多项目对比；
- 年度现金流模型；
- 融资结构、税费、退出估值和 IRR；
- 工程造价清单级别的成本测算。

当前版本的价值是把产品方向、输入框架、计算口径和输出形态先跑通。

## 如何运行

这是一个静态网页原型，可以直接用浏览器打开：

```text
index.html
```

如果需要用本地服务运行，也可以在项目目录执行：

```bash
python3 -m http.server 8000
```

然后打开：

```text
http://localhost:8000
```
